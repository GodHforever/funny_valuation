# earnings-insight — 公司财报深度研判与前瞻分析

## 核心理念

> 研判不是预测未来，而是**基于当前信号，评估不同情景发生的概率与影响**。好的研判应当是有条件的、可证伪的，而不是模糊的乐观或悲观表态。

## 使用流程

### Phase 1: 数据获取（自动）

根据股票代码自动判断市场并获取数据：

**A股（6位纯数字）：**
```bash
python {skill_path}/scripts/data_fetcher.py --code {股票代码} --output-dir {用户指定路径}
```

**港股/美股（使用 ticker-cli）：**
```bash
# 港股（代码.HK格式，如 0700.HK）
npx ticker-cli quote {代码}
npx ticker-cli summary {代码}
npx ticker-cli chart {代码} -p 1y
npx ticker-cli indicator {代码} sma -p 1y
npx ticker-cli news {代码} --limit 10
npx ticker-cli insights {代码}
npx ticker-cli recommendations {代码}

# 美股（纯字母，如 AAPL）
# 同上命令，替换代码即可
```

**市场识别规则：**
- 纯数字6位 → A股（自动判断沪深）
- 数字.HK → 港股
- 纯字母/含数字的字母组合 → 美股

### Phase 2: 补充信息（交互）

数据获取完成后，引导用户提供年报/财报中的非结构化信息。**必须告知用户可以通过三种方式提供：**

#### 方式A：对话输入
分步引导用户提供以下内容（按重要性排序）：

1. **管理层讨论与分析（MD&A）** — 最关键，第二/六层分析的核心输入
2. **业务发展概况 / 主营业务描述** — 第三层行业分析的输入
3. **风险因素** — 第五层风险前瞻的输入
4. **重大事件 / 战略方向** — 战略可执行性评估的输入
5. **（可选）前次分析报告路径** — 用于跟踪清单对比

提示用户：
```
请提供以下财报内容（可直接粘贴文本，或提供文件/目录路径）：

1. 管理层讨论与分析（MD&A）[必需]
2. 业务发展概况 / 行业描述 [建议]
3. 风险因素 [建议]
4. 战略方向 / 重大事项 [可选]
5. 上次分析报告路径 [可选，用于对比]

提示：如果已用 earnings-report-extractor 提取过年报，直接提供目录路径即可。
```

#### 方式B：文件路径
用户提供单个文件路径 → 直接读取内容

#### 方式C：目录路径
用户提供目录路径（通常为 earnings-report-extractor 的提取输出）→ 自动处理：

1. `ls` 列出目录下所有文件
2. 如存在 `index.md`，优先读取了解目录结构
3. 按文件名和内容自动识别各章节用途，映射到分析框架：

| 文件名关键词 | 映射到分析层 |
|-------------|-------------|
| 管理层讨论、经营情况讨论、MD&A | 第二层(业务批判性解读) + 第六层(战略评估) |
| 风险、风险因素 | 第五层(风险前瞻识别) |
| 业务概要、主营业务、业务描述 | 第三层(行业与竞争格局) |
| 公司治理、董事会 | 第六层(激励机制/治理分析) |
| 重要事项、重大事件 | 第五层(已知风险) + 第四层(交叉验证) |
| 财务报告、财务报表附注 | 第一层(数据拆解补充) |
| 董事长致辞、总裁致辞 | 第二层(管理层表述分析) |
| 股份变动、股东情况 | 第六层(股东利益一致性) |

4. 自动过滤低价值章节（环境责任报告、投资者关系等），但保留索引备查
5. 读取映射到的关键章节内容，融入后续分析

### Phase 3: 历史报告对比（可选）

如果用户提供了前次分析报告路径：
1. 读取该报告的"结构化结论"和"跟踪检查清单"部分
2. 逐条验证前次研判：已验证/已证伪/待观察
3. 在新报告中增加"上次研判回顾"章节

### Phase 4: 七层分析（自动）

按以下框架逐层生成分析。每一层的产出为下一层的输入。

### Phase 5: 存储

询问用户希望保存报告的路径，将完整 Markdown 报告保存到指定位置。

## 七层分析框架

### 第一层：财务数据深度拆解

**数据来源：** data_fetcher.py 自动获取的结构化JSON / ticker-cli 输出

**分析要点：**

1. **收入质量分析**
   - 营收增速与行业增速对比（是超越行业还是随波逐流？）
   - 营收结构如有披露（产品/地区/客户），判断集中度风险
   - 营收与应收账款增速对比（应收增速>营收增速 = 放宽信用/回款困难信号）

2. **利润质量分析**
   - 归母净利润 vs 扣非归母净利润 → 非经常损益占比
   - 毛利率/净利率趋势 → 是扩张还是收缩？
   - 费用率变化 → 研发/销售/管理费用的占比趋势
   - 减值损失 → 资产减值+信用减值金额及变化

3. **资产负债结构**
   - 资产负债率、流动比率、速动比率
   - 有息负债规模及净现金状况
   - 资产结构：重资产vs轻资产，商誉占比
   - 杜邦分析拆解ROE来源

4. **现金流健康度**
   - 现金流类型判断（八种经典类型）
   - 收现比（销售收现/营收）→ 收入变现能力
   - 经营现金流/净利润 → 盈利含金量
   - 自由现金流估算（经营现金流-资本支出）

### 第二层：业务表述批判性解读

**数据来源：** 用户提供的管理层讨论、董事长致辞等

**核心原则：管理层的"行动"比"语言"更能反映其真实判断。**

**分析要点：**

1. **语言与行动的一致性检验**
   - 管理层说"对前景充满信心" → 但是否在增加资本支出？扩大核心团队？
   - 管理层说"积极拓展新业务" → 研发投入是否真的在增加？
   - 管理层强调"成本控制" → 费用率是否真的在下降？

2. **被淡化或回避的问题**
   - 年报中哪些重要话题被一笔带过？
   - 与上年年报对比，哪些曾经强调的方向不再提及？
   - 风险因素部分是否有新增的重要风险？

3. **定量vs定性的切换**
   - 管理层在成绩好的领域用数字，在差的领域用形容词 → 刻意模糊化
   - 用"基本完成"、"稳步推进"等模糊表述替代具体进展 → 可能进展不顺

### 第三层：行业与竞争格局

**数据来源：** data_fetcher.py 行业对比数据 + 用户提供的业务描述

**分析要点：**

1. **行业周期定位**
   - 当前行业处于周期哪个阶段（复苏/繁荣/衰退/萧条）？
   - 行业平均利润率处于历史什么水平？均值回归压力？
   - 行业资金流向是流入还是流出？

2. **竞争坐标**
   - 公司在同行中的排名（市值、PE、PB、ROE）
   - 公司估值相对行业平均的溢价/折价程度
   - 市场份额变化趋势（如有数据）

3. **竞争格局变化**
   - 新进入者威胁
   - 替代品渗透速度
   - 供应链关系变化

### 第四层：交叉验证

将前三层发现进行**一致性检验**：

1. **财务数据 vs 业务表述**
   - 管理层声称业务增长强劲 ↔ 实际营收是否增长？
   - 管理层声称降本增效 ↔ 费用率是否真的下降？
   - 管理层强调研发投入 ↔ 研发费用占比是否提升？

2. **信号分类汇总**
   - **高可信度信号**：已发生，数据可验证（财务数据、已完成的交易等）
   - **中可信度信号**：趋势性，有迹可循（领先指标变化趋势、行业数据等）
   - **低可信度信号**：管理层表述，需交叉验证（战略规划、前景展望等）
   - **假设性信号**：情景推演，需明确标注

3. **矛盾识别**
   - 列出发现的所有信号矛盾，评估哪一方更可信

### 第五层：前瞻研判

**这是整个框架最核心的层次。**

#### 5.1 领先指标分析

使用 data_fetcher.py 提取的领先指标，判断未来业绩方向：

| 领先指标 | 变化方向 | 可能预示 |
|---------|---------|---------|
| 合同负债+预收账款 | 持续增长 | 未来收入有保障 |
| 合同负债+预收账款 | 突然下降 | 新订单获取能力减弱 |
| 在建工程 | 大幅增加 | 管理层真实看好未来（行动信号） |
| 资本支出 | 扩产性增长 | 对需求的真实判断 |
| 研发费用占比 | 持续提升 | 未来竞争力可能增强 |
| 应付职工薪酬 | 大幅增长 | 可能在扩张关键人才 |
| 应收账款/存货 | 增速>营收增速 | 可能暗示需求放缓或回款困难 |

#### 5.2 三情景分析

**必须输出三个情景，每个情景必须包含具体条件、概率评估和证伪条件。**

```
基准情景（最可能发生，概率 X%）
    ├── 核心假设：[具体的、可验证的假设条件]
    ├── 收入/利润路径：[具体的演变逻辑和量化估算]
    ├── 支撑证据：[来自前四层的数据依据，标注信号可信度]
    └── 证伪条件：[什么情况出现意味着此情景不成立]

乐观情景（需要额外条件，概率 Y%）
    ├── 触发条件：[什么因素需要超预期]
    ├── 潜在收益空间：[业绩弹性有多大]
    └── 证伪条件：[什么情况出现意味着乐观预期不成立]

悲观情景（下行风险，概率 Z%）
    ├── 风险触发点：[哪些因素是最大威胁]
    ├── 下行幅度评估：[损失有多严重]
    └── 早期预警信号：[如何提前识别]
```

**质量要求：**
- 每个情景的条件必须**具体**，不能是"如果市场好转"
- 三个情景的概率之和应为100%
- 必须指明什么时候情景会被**证伪**

#### 5.3 增长可持续性评估

诊断当前增长的驱动类型：

| 驱动类型 | 判断逻辑 |
|---------|---------|
| 渗透率驱动 | 判断天花板距离 |
| 价格提升驱动 | 判断定价空间 |
| 份额抢占驱动 | 判断竞争可持续性 |
| 行业扩容驱动 | 判断行业周期位置 |
| 并购整合驱动 | 判断整合质量与后续标的 |

**评估要点：**
- 第二增长曲线是战略愿景还是已有实质进展？
- 规模效应是否真实存在？（成本是否随规模下降？）
- 当前利润率处于历史高位/低位？均值回归压力？

#### 5.4 风险前瞻识别

**分层体系：**

```
已知风险（财报已披露）→ 评估应对方案是否足够
隐含风险（数据可推断但未披露）→ 为什么没披露？影响多大？
系统性风险（行业/宏观，公司不可控）→ 暴露程度？对冲能力？
尾部风险（低概率高影响）→ 触发条件？冲击量级？
```

**重点前瞻的风险类型：**
- 商业模式风险：核心客户依赖、技术替代、监管冲击
- 财务风险前瞻：债务到期结构、利率敏感性、汇率敞口
- 竞争风险演变：新进入者弹药、竞争对手转向、替代品渗透
- 执行风险：战略转型历史成功率、组织能力匹配度

**"被忽视的风险"专项：** 专门扫描哪些重要风险在财报中被一笔带过、轻描淡写或未提及。

#### 5.5 战略可执行性评估

| 评估维度 | 核心问题 |
|---------|---------|
| 资金充裕度 | 战略所需资本支出，现金流能否支撑？ |
| 时间窗口 | 行业窗口期与战略落地周期是否匹配？ |
| 历史执行力 | 过去战略目标实现率如何？ |
| 激励一致性 | 管理层薪酬考核与长期战略是否绑定？大股东利益是否一致？ |

### 第六层：结构化结论

**输出必须包含以下全部字段：**

```
【核心判断】
用1-2句话说明最重要的前瞻性结论

【信号可信度】
标注此判断基于的最高信号层级（高/中/低可信度/假设性）

【判断依据】
列出3-5个最关键的支撑证据（必须来自财报数据或可验证信息）

【主要假设】
明确列出这个判断成立所依赖的核心假设

【证伪条件】
什么情况出现，意味着这个判断是错的

【关键观察指标】
未来应当重点跟踪哪些指标来验证判断？时间节点是什么？

【不确定性说明】
判断的置信度？哪些方面信息不足？
```

### 第七层：跟踪检查清单

在报告末尾生成可检验的跟踪清单：

### 情景触发/证伪条件监控

- [ ] 乐观情景触发条件：[具体条件]
- [ ] 悲观情景预警信号：[具体条件]

### 风险监控项
- [ ] [风险1] — 监控指标：[X]

## 输出规范

### 报告结构模板

参见 `{skill_path}/references/analysis_template.md`

### 信号标注规范

每个重要结论必须在行尾标注信号可信度：

- `[高可信度]` — 基于已发生的、可验证的数据
- `[中可信度]` — 基于趋势性数据，有迹可循
- `[低可信度]` — 基于管理层表述，需交叉验证
- `[假设性]` — 基于情景推演，需明确标注

### 禁止的表述模式

以下表述模式在分析报告中**严格禁止使用**：

| 禁止的表述 | 原因 | 应替换为 |
|-----------|------|---------|
| "公司未来前景光明" | 无法验证 | 具体说明哪些指标支持积极判断 |
| "市场存在不确定性" | 废话 | 具体说明哪些不确定性最关键 |
| "如果战略顺利推进" | 假设过于宽泛 | 明确"顺利"的具体条件 |
| "长期来看值得关注" | 模糊时间框架 | 给出具体判断时间窗口（如"未来2-3个季度"） |
| 直接给出买入/卖出建议 | 不合规 | 给出多情景下的业绩路径 |
| "整体而言" + 模糊结论 | 回避具体判断 | 给出具体的、可证伪的结论 |
| "有望实现" | 概率不明 | 给出概率评估和实现条件 |
| "建议关注" | 无行动指引 | 给出具体观察指标和判断阈值 |

### 免责声明

每份报告末尾必须包含：

> 本报告基于公开财务数据和年报信息，运用结构化分析框架生成，仅供研究参考，不构成任何投资建议。报告中的前瞻性研判基于特定假设条件，实际结果可能与预期存在重大差异。投资者应当独立判断，自行承担投资风险。

## 港股/美股补充说明

由于港股/美股的财务数据通过 `ticker-cli` 获取，数据颗粒度可能不如A股东方财富API详细。在分析时：

1. **领先指标**：可能无法自动提取合同负债等明细，需依赖用户提供的年报信息补充
2. **行业对比**：使用 `ticker-cli recommendations` 和 `ticker-cli insights` 获取分析师观点作为参考
3. **估值对比**：通过 `ticker-cli summary` 获取核心估值指标
4. **分析框架不变**：七层分析框架完全适用，只是数据来源和获取方式不同

## 调用模式

### Mode A: 独立运行
用户直接调用 `/earnings-insight`。Skill 自行获取数据并执行七层分析，直接输出给用户。

### Mode B: 协同运行（工作流调用）
工作流调用，结构化数据输入输出。

**数据获取：**
```bash
python .claude/skills/components/earnings-insight/scripts/data_fetcher.py \
  --code {stock_code} --mode collaborative --data-dir data/{stock_code}/
```

**输入：**
- 提取的章节文件: `data/{stock_code}/earnings-extracted/{code}_第X节_*.md`（可选，用于 Layer 2 批判性阅读）

**输出：**
- 金融数据: `data/{stock_code}/earnings-analysis/{code}_insight_data.json`
- stock_info 缓存: `data/{stock_code}/market-data/stock_info.json`
- 深度研判报告: `data/{stock_code}/earnings-analysis/{code}_{name}_深度研判报告.md`

**输出契约:** `specs/contracts/financial-statements.json`

**质量门：**
```bash
python scripts/validate_contract.py \
  --schema specs/contracts/financial-statements.json \
  --data data/{stock_code}/earnings-analysis/{code}_insight_data.json
```
