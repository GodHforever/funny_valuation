# Report Generator — 统一报告生成器

## 功能概述

**纯 Prompt 驱动**，无 Python 脚本。读取 `data/{code}/analysis/` 下的分析产物，按标准模板生成报告。

### 报告模板

| 模板 | 页数 | 适用场景 |
|------|------|---------|
| **快速估值报告** | 1-2 页 | 快速了解估值水平和安全边际 |
| **深度研究报告** | 5-10 页 | 全面基本面研究 + 估值 + 风险 |
| **行业对比报告** | 3-5 页 | 行业内公司对比排名 |
| **风险扫描报告** | 1 页 | 风险指标速览 |

## 强制规范

### 数据来源标注
每个数据结论必须标注来源：
```
营收增速 25.3%[数据来源: earnings-insight, 2026-05-12]
```

### 信号可信度标注
每个分析结论必须标注信号层级：
```
公司处于成长期[可信度: 高 — 基于3年营收CAGR>20%]
行业景气度可能回升[可信度: 中 — 基于PMI连续2月回升趋势]
明年出口收入可能受汇率影响[可信度: 低 — 假设性推演]
```

可信度分级：
- **高**：基于已发生的财务数据或可验证的事实
- **中**：基于可观测的趋势或多数据交叉验证
- **低**：基于情景推演或管理层表述
- **假设性**：明确的假设条件，需要证伪

### 免责声明
所有报告末尾必须包含：
```
---
**免责声明**: 本报告由 AI 辅助生成，基于公开数据分析，不构成投资建议。
数据来源为东方财富等公开 API，可能存在延迟或误差。投资有风险，决策需谨慎。
```

## 使用方法

### 执行流程

#### Step 1: 确认分析产物
检查 `data/{code}/analysis/` 下有哪些分析文件可用。

#### Step 2: 选择模板
根据可用数据和用户需求选择报告模板。

#### Step 3: 生成报告
读取分析产物，按模板结构组织报告，写入 `data/{code}/reports/`。

### 输入
- `data/{code}/analysis/` — 所有分析中间产物
- `data/{code}/cache/stock_info.json` — 基础行情数据

### 输出
- `data/{code}/reports/{code}_{报告类型}_{YYYYMMDD}.md`

## 调用模式

### Mode A: 独立运行
用户指定股票代码和报告类型，Skill 检查可用数据后生成报告。

### Mode B: 协同运行（工作流调用）
工作流完成分析阶段后调用，自动选择合适的报告模板。
