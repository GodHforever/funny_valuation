当前的仓库是一个私人的，定制化的金融分析工具仓库，通过agent和流水线的设定，形成了一个闭环的，可控性高，稳定的流程，基于各种skills能够完成各种任务。但需要重构。
# 问题分析
1. 当前流水线设计过于死板，无法根据特定任务充分组合各种skills。我认为可以定制针对特定细分任务的agent，同时设计通用agent来统一对agent和skills进行调度。
2. 分析功能的严谨性和深度不够，很多时候没有考虑当前宏观行业基本面，企业本身的经营特点和独有特性。
3. 数据，结果，文档的生成和存储方式不统一，不严谨，需要充分规定和限制，做到严格控制

# 核心要求
1. 充分借鉴dexter的设计思想和特点，借鉴其可用之处，改造当前仓库
2. 确保严格的，可闭环的，可控的，严谨的分析流程设计，充分利用软约束和硬约束的方式
3. 确保数据来源的真实性，严谨性，言之有物
4. 尽可能通过免费途径，借鉴已有的skills充分优化和添加新的skills
5. 仍然以skills和agent编排为主，借助claude code和其他agent工具运行。 不需要把项目设计为一个完整的llm agent
